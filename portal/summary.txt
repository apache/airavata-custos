Bug fixes during the session
============================

- Unnecessary font tweak: my first globals.css rewrite added a --font-sans
  @theme entry. The user flagged this — original config had no theme
  customizations to migrate. Reverted to plain @import "tailwindcss" + the
  existing body Inter declaration. (Also saved a memory entry about this.)

- Tailwind v4 version mismatch: first install resolved @tailwindcss/postcss
  to 4.0.0 while tailwindcss was 4.3.0, causing
  "Missing field `negated` on ScannerOptions.sources" at build time. Fixed
  by pinning both to @latest (4.3.0).

- IDE-only TS error on `import "./globals.css"` (TS2882): added
  src/types/css.d.ts with `declare module "*.css";`. tsc had been passing
  already; this just silences the editor diagnostic.

- NextAuth MissingSecret 500s on /api/auth/* : .env.local didn't exist.
  Generated a fresh 32-byte secret (node crypto, not the Vault token from
  the signer config — intentionally not reused) and wrote .env.local with
  NEXTAUTH_URL=http://127.0.0.1:5173 (the portal dev port from package.json,
  NOT the :3000 from the example template).

- NextAuth InvalidEndpoints 500s : OIDC_ISSUER_URL was empty. Per your
  choice, added a dev-mode fallback in auth.ts: when OIDC_ISSUER_URL is
  unset it registers a Credentials provider (id "dev") whose authorize()
  returns a fixed Dev User. The session callback synthesizes
  accessToken="dev-token" in that mode, which the signer accepts when run
  with DEV_MODE=true. Setting OIDC_ISSUER_URL automatically deactivates the
  fallback and the OIDC provider takes over.


Things to be aware of
=====================

- portal/.env.local now contains a real NEXTAUTH_SECRET. It's covered by
  the updated .gitignore — do NOT commit it. Rotate the secret if it has
  appeared in any shared transcript or screenshot.

- Dev-mode auth is automatic when OIDC_ISSUER_URL is empty. On first load
  you'll see a single-button "Sign in with Dev" page (NextAuth's default
  sign-in screen) since I used signIn() without a provider id. If you want
  silent auto-sign-in (no click), say the word — small change in
  PortalLayout to call signIn("dev", { redirect: false }).

- The signer must be started with DEV_MODE=true to accept the "dev-token"
  bearer the portal sends in dev mode. See
  extensions/SSH-Certificate-Signer/README.md.

- Figma design (shared mid-session) is saved as a memory reference, but I
  did NOT restyle the existing PortalLayout/CertificateListPage to match
  the Figma's "CUSTOS" sidebar/header chrome — that wasn't in the original
  task list. Happy to do a visual pass next.

- shadcn init added a Geist font import in layout.tsx and a font-sans @layer
  rule on <html>. The body keeps an explicit `font-family: Inter, ...` rule,
  so the visible font is still Inter (specificity wins). If you'd rather
  fully drop Geist, the layout.tsx Google-font import can come out.

- hooks.test.tsx still exercises useUserInfo() and the /api/v1/userinfo
  call. The proxy now requires a session, so in dev these tests pass only
  because they mock fetch directly. Real /api/v1/userinfo calls from
  PortalLayout are gone — kept the test + hook in case other code uses
  them later.

- RevokeDialog dropped triggerClassName/confirmClassName/cancelClassName.
  If any other code paths passed those (none found via grep), they'll
  silently no-op now.

- next lint emits a deprecation notice ("will be removed in Next.js 16,
  migrate to ESLint CLI"). Not blocking, but worth a separate cleanup PR.
